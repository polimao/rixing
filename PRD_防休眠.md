> 所属产品：日行 · RiXing(macOS 菜单栏待办 + 日历)
> 文档版本：v0.2(按 ToDesk 式语义修订)
> 撰写视角：产品经理
> 状态：评审中

---

## 0. 修订说明(v0.2)

v0.1 把「保持屏幕常亮」当作默认必选项,与真实需求相悖。经确认,本功能的**真实目标是对齐 ToDesk / Amphetamine 这类「防休眠」**:

> **系统不要休眠(空闲休眠被阻止),但显示器允许正常熄屏、允许锁屏;目的是让后台持续运行的程序(编译、下载、渲染、远程会话、同步等)不被系统中断。**

因此本版:
- **删除「保持屏幕常亮」开关**。显示器照常可以息屏、可以锁屏——这是预期行为,不是 bug。
- 防休眠只阻止「系统空闲休眠」(`caffeinate -i`),不阻止「显示器休眠」(不用 `-d`)。
- 为兼容笔记本离电场景,**不使用 `-s`**(否则仅接电源时生效),即电池下也生效。
- 保留时长档位、到点自动恢复、轻量通知、退出兜底。

## 1. 背景与目标

### 1.1 背景
macOS 会在用户闲置一段时间后自动休眠。但很多场景下用户**不希望系统睡着**,尤其是不希望**后台任务被中断**:

- 下载 / 上传大文件、跑脚本、渲染、编译时,离开一会儿回来发现任务因系统休眠中断;
- 远程控制 / 保持在线状态(类似 ToDesk)时,不希望因休眠掉线;
- 后台同步、定时任务、长时运算期间,需要系统持续在线。

> 注意:本功能**不要求屏幕一直亮着**。用户离开时屏幕可以正常熄屏、可以锁屏,只要「系统别睡」即可——这正是「防休眠」与「防熄屏」的区别。

系统偏好设置里改「节能」很繁琐,且改完容易忘记调回。市面上 Amphetamine、Caffeine、KeepingYouAwake、ToDesk 的防休眠正是解决这个高频痛点。

日行本身**常驻菜单栏、随手可用、无 Dock 干扰**,天然是承载「临时防休眠」这类轻量系统工具的最佳容器。

### 1.2 目标
- **一键防休眠**:让用户在托盘 / 设置里一步开启 / 关闭防休眠。
- **可控时长**:支持设定持续时长,到点自动恢复系统默认行为,避免「忘记关」。
- **符合直觉的默认值**:开箱即用,不配置也能用。
- **零学习成本**:交互与现有设置(主题、快捷键、分屏)保持一致风格。
- **不打扰显示**:开启后屏幕照常熄屏 / 锁屏(后台程序不受影响)。

### 1.3 非目标(本期不做)
- 不做「按 App / 进程条件触发」的自动化。
- 不做 Windows / Linux 端(当前产品仅 macOS)。
- 不做「防休眠计划表 / 定时段」。
- 不接管系统电源策略,不修改用户的系统节能设置(只用临时断言,退出即恢复)。
- **不做「保持屏幕常亮」**(v0.1 曾列入,经确认与需求相悖,已移除)。

## 2. 用户与场景

| 用户角色 | 典型场景 | 期望 |
| --- | --- | --- |
| 效率 / 开发用户 | 跑长任务(下载、编译、备份) | 设一个较长时长,任务期间系统不睡,结束自动关 |
| 远程 / 在线用户 | 远程控制、保持会话(类 ToDesk) | 系统持续在线不被中断,屏幕可熄屏 |
| 通用用户 | 临时离开但要保持后台运行 | 快速开一会儿,不用记得关 |

### 核心用户故事
- 作为用户,我想**点一下就开启防休眠**,这样我离开时后台程序不会因系统休眠而中断。
- 作为用户,我想**选择保持多久**(如 1 小时),到点自动恢复,这样我不必担心忘记关掉而费电。
- 作为用户,我想有一个**默认时长**,大多数时候不用选,点开即用。
- 作为用户,我想**一眼看出现在是否处于防休眠状态**、还剩多久。
- 作为用户,我**接受屏幕照常熄屏 / 锁屏**——我只需要「系统别睡」。

## 3. 功能需求

### 3.1 功能总览
防休眠功能由三部分组成:
1. **托盘菜单开关**(主入口,快速操作,**建议实现**);
2. **设置页配置项**(默认时长、状态展示);
3. **状态反馈**(剩余时间、到点自动恢复 + 轻量通知)。

### 3.2 设置页配置(设置 → 「防休眠」分区)

| 配置项 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 默认时长 | 下拉 / 滑块 | **1 小时** | 决定开启防休眠时默认撑多久;可选 10 分 / 30 分 / 1 时 / 2 时 / 4 时 / 12 时 / 1 天 / 3 天 / 一周 / 一直保持 |
| 开机时自动开启防休眠 | 开关 | 关 | 随 App 启动即进入防休眠(配合「开机自启动」) |

> **已移除 v0.1 的「保持屏幕常亮」开关**:本功能语义本就不阻止显示器休眠,该开关无意义。

> 关于「默认时长为什么定 1 小时」:15 分钟太短、覆盖不了多数「离开一下」的场景;一直保持又太激进、容易忘关费电。1 小时是「够用 + 有兜底」的平衡点。此默认值需在评审中确认。

### 3.3 行为定义
- **未开启时**:开关关闭,系统按常规策略休眠。
- **已开启时**:系统不进入空闲休眠(后台程序持续运行),但**显示器可正常熄屏、可锁屏**。菜单 / 设置中显示剩余时间(如「防休眠 · 剩 42 分钟」)。
- 选「一直保持」= 无限时长,直到用户手动关闭或退出 App。
- **快捷开关**:再次点击当前已勾选项 = 关闭(可选实现,降低操作成本)。

### 3.4 状态反馈
- **托盘图标**:防休眠开启时,托盘图标叠加一个轻微状态标识(小圆点 / 咖啡杯角标),让用户余光可辨。需与现有极简托盘风格统一。
- **剩余时间**:菜单 / 设置中实时显示剩余时长(每分钟刷新即可)。
- **到点行为**:计时结束后自动解除防休眠、恢复系统默认,并给一次**轻量通知 / Toast**(如「防休眠已结束,电脑已恢复常规休眠策略」),避免用户不知何时恢复。
- **手动关闭**:可随时关闭,立即生效,屏幕状态不受影响。

### 3.5 与现有功能的关系
- **番茄钟专注模式**:进入番茄钟时,专注期间**建议自动开启防休眠**(至少保持系统不睡),专注结束后恢复到用户此前的防休眠状态。P1。
- **窗口失焦自动隐藏**:防休眠与窗口显隐彼此独立,互不影响。

## 4. 交互与状态流转

### 4.1 状态机
```
        点击时长项 / 默认开启
[关闭] ─────────────────────────▶ [已开启(计时中)]
   ▲                                    │
   │  点「关闭」/ 计时到点 / 退出 App      │  切换其他时长 = 重置计时
   └────────────────────────────────────┘
                                         │
                          选「一直保持」 ▼
                              [已开启(无限,手动关闭)]
```

### 4.2 关键规则
- 切换时长(如从 15 分钟改到 1 小时)= **重置计时**,以新时长重新开始。
- 「一直保持」无倒计时,仅手动关闭或退出 App 时解除。
- **退出 App 必须解除所有电源断言**,不得残留导致电脑永不休眠(重要:资源安全)。
- 系统睡眠被用户主动触发(如合盖、手动睡眠)时,遵循系统行为,不强行阻止合盖睡眠。
- **显示器熄屏 / 锁屏均不视为「防休眠失效」**,这是预期行为。

### 4.3 时长计算的准确性(实现要点)
- 防休眠的「时长」以**开启时刻(startedAt)**为基准。
- 中途退出 / 重启 App 后恢复时,断言时长必须按**剩余时间**计算,而不是重新撑满整段:
  - 例:设 1 小时,用了 30 分钟后重启 → 恢复后只应再撑约 30 分钟,而非再撑 1 小时。
- 后端 `caffeinate -t <剩余秒数>` 与前端倒计时共用同一 `startedAt`,保证两端一致。

## 5. 技术实现

> 本节为产品侧的实现方向建议,最终以研发评估为准。

- **原理**:使用 macOS 电源断言,通过系统自带 `caffeinate` 子进程接管:
  - 阻止系统空闲休眠:`caffeinate -i`(`kIOPMAssertionTypePreventUserIdleSystemSleep`)——后台程序持续运行的关键;
  - 合盖( clamshell )也保持:`caffeinate -m`;
  - **不使用 `-d`**:`-d` 会阻止显示器休眠,与「允许熄屏」需求相悖,**绝不添加**;
  - **不使用 `-s`**:`-s` 仅接电源时生效,会导致笔记本离电时失效,**不添加**以保证电池下也生效。
- Tauri(Rust)后端持有 `caffeinate` 子进程句柄,前端 / 托盘通过命令开启、关闭;倒计时由后端 `-t` 与前端计时器共同管理(以剩余秒数为准),到点自动释放断言。
- **务必**在 App 退出(`RunEvent::Exit`)与切换时释放子进程(`kill` + `wait` 回收),防止系统永不休眠或僵尸进程残留。
- 配置随现有设置一起以 JSON 持久化(与 `.todos_settings.json` 同机制):`keepAwake = { enabled, index, startedAt }`;运行态(是否开启、剩余时间)不额外持久化 `startedAt` 之外的字段,App 重启后除非「开机自动开启」否则默认关闭。
- 不调用需要额外权限的接口;防休眠断言无需辅助功能 / 特殊授权。

## 8. 边界与异常

| 情况 | 预期处理 |
| --- | --- |
| App 崩溃 / 强退 | 断言随 `caffeinate` 进程结束失效,系统恢复正常休眠(注:异常退出时建议依赖进程组被杀;正常退出有 `RunEvent::Exit` 兜底) |
| 用户手动合盖 | 遵循系统行为,防休眠不阻止合盖睡眠 |
| 同时开启多个时长 | 以最后一次选择为准,重置计时 |
| 显示器熄屏 / 锁屏 | **正常,非异常**;系统仍不休眠,后台程序继续 |
| 电池模式(未接电源) | 正常生效(`-i` 不受电源限制) |
| 自定义时长非法(超范围 / 非数字) | 拦截并提示,恢复上一次有效值 |

## 9. 验收标准(Acceptance Criteria)

1. 开启防休眠后,系统在所选时长内**不进入空闲休眠**,后台程序不被中断。
2. 开启防休眠后,**显示器可正常熄屏、可锁屏**(屏幕状态不受防休眠影响)。
3. 选定时长后,到点自动恢复系统默认休眠,并有轻量通知。
4. 中途退出 / 重启 App,恢复后的剩余时长与「开启时设定的剩余时间」一致(不会重新撑满整段)。
5. 退出 App 后,系统休眠行为完全恢复正常(无残留断言、无僵尸进程)。
6. 「一直保持」下无倒计时,仅手动关闭 / 退出时解除。
7. 深色 / 浅色主题、多语言下界面与文案均正常。
