# 防休眠(Keep Awake)功能 · 产品需求文档 (PRD)

> 所属产品：日行 · RiXing(macOS 菜单栏待办 + 日历)
> 文档版本：v0.1(需求评审稿)
> 撰写视角：产品经理
> 状态：待评审

---

## 1. 背景与目标

### 1.1 背景
macOS 会在用户闲置一段时间后自动息屏、休眠。但在很多场景下用户**不希望电脑睡着**:

- 下载 / 上传大文件、跑脚本、渲染、编译时,离开一会儿回来发现任务因休眠中断;
- 看文档、看视频、演示、投屏时,不想频繁被息屏打断;
- 远程连接 / 保持在线状态时,不想因休眠掉线。

系统偏好设置里改「节能」很繁琐,且改完容易忘记调回,长期不休眠又费电。市面上 Amphetamine、Caffeine、KeepingYouAwake 等独立工具正是解决这个高频痛点。

日行本身**常驻菜单栏、随手可用、无 Dock 干扰**,天然是承载「临时防休眠」这类轻量系统工具的最佳容器——用户不需要再多装一个 App。

### 1.2 目标
- **一键防休眠**:让用户在托盘上一步开启 / 关闭防休眠。
- **可控时长**:支持设定持续时长,到点自动恢复系统默认行为,避免「忘记关」。
- **符合直觉的默认值**:开箱即用,不配置也能用。
- **零学习成本**:交互与现有设置(主题、快捷键、分屏)保持一致风格。

### 1.3 非目标(本期不做)
- 不做「按 App / 进程条件触发」的自动化(如「检测到某程序运行时自动防休眠」)。
- 不做 Windows / Linux 端(当前产品仅 macOS)。
- 不做「防休眠计划表 / 定时段」。
- 不接管系统电源策略,不修改用户的系统节能设置(只用临时断言,退出即恢复)。

## 2. 用户与场景

| 用户角色 | 典型场景 | 期望 |
| --- | --- | --- |
| 效率 / 开发用户 | 跑长任务(下载、编译、备份) | 设一个较长时长,任务期间不休眠,结束自动关 |
| 内容 / 演示用户 | 看视频、做演示、投屏 | 保持屏幕常亮,不息屏 |
| 通用用户 | 临时离开但要保持在线 | 快速开一会儿,不用记得关 |

### 核心用户故事
- 作为用户,我想**点一下托盘就开启防休眠**,这样我离开时电脑不会睡着。
- 作为用户,我想**选择保持多久**(如 1 小时),到点自动恢复,这样我不必担心忘记关掉而费电。
- 作为用户,我想有一个**默认时长**,大多数时候不用选,点开即用。
- 作为用户,我想**一眼看出现在是否处于防休眠状态**、还剩多久。

## 3. 功能需求

### 3.1 功能总览
防休眠功能由三部分组成:
1. **托盘菜单开关**(主入口,快速操作);
2. **设置页配置项**(默认时长、防休眠等级、状态展示);
3. **状态反馈**(托盘图标 / 文案、剩余时间、到点自动恢复)。

### 3.2 托盘菜单(主入口)
在托盘右键菜单中新增「防休眠」区块:

```
┌─────────────────────────────┐
│  ☕ 防休眠                     │
│    ○ 关闭                     │   ← 当前状态可勾选
│    ● 15 分钟                  │
│    ○ 30 分钟                  │
│    ○ 1 小时                   │
│    ○ 2 小时                   │
│    ○ 5 小时                   │
│    ○ 一直保持(手动关闭)       │
│    ─────────────             │
│    ○ 自定义时长…              │
└─────────────────────────────┘
```

行为定义:
- **未开启时**:菜单项显示为「防休眠」,点击子项即以对应时长开启。
- **已开启时**:菜单顶部显示剩余时间(如「防休眠 · 剩 42 分钟」),当前生效的时长项打勾;选择「关闭」立即结束。
- 选「一直保持」= 无限时长,直到用户手动关闭或退出 App。
- 「自定义时长…」弹出输入(分钟数,范围 1–1440)。
- **快捷开关**:再次点击当前已勾选项 = 关闭(可选实现,降低操作成本)。

> 设计取舍:考虑到该 App 是菜单栏原生、无主窗承载此功能,**托盘菜单是唯一必需的入口**;设置页负责「默认值与偏好」,不承担高频切换。

### 3.3 设置页配置(设置 → 新增「防休眠」分区)
在设置窗口左侧标签或「通用」页内新增「防休眠」相关配置(建议独立分区,图标沿用现有 tab 风格):

| 配置项 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 默认时长 | 下拉选择 | **1 小时** | 决定点击托盘「防休眠」主项时默认开启多久;可选 15 分钟 / 30 分钟 / 1 小时 / 2 小时 / 5 小时 / 一直保持 / 自定义 |
| 自定义默认时长 | 数字输入(分钟) | — | 仅当默认时长选「自定义」时可编辑,范围 1–1440 |
| 保持屏幕常亮 | 开关 | **开** | 开=同时阻止息屏(display sleep);关=仅阻止系统休眠,屏幕仍可正常息屏但系统不睡 |
| 开机时自动开启防休眠 | 开关 | 关 | 随 App 启动即进入防休眠(配合「开机自启动」) |
| 播放轻听音乐时自动防休眠 | 开关 | 关 | 与现有「轻听」功能联动,播放期间自动保持不休眠(可选,见 §3.6) |

> 关于「默认时长为什么定 1 小时」:15 分钟太短、覆盖不了多数「离开一下」的场景;一直保持又太激进、容易忘关费电。1 小时是「够用 + 有兜底」的平衡点,且用户可随时改。此默认值需在评审中确认。

### 3.4 状态反馈
- **托盘图标**:防休眠开启时,对应托盘图标叠加一个轻微的状态标识(如小圆点 / 眼睛睁开 / 咖啡杯角标),让用户余光可辨。需与现有透明托盘图标风格统一,不破坏简洁感。
- **剩余时间**:托盘菜单中实时显示剩余时长(每分钟刷新即可)。
- **到点行为**:计时结束后自动解除防休恢复系统默认,并给一次**轻量通知 / Toast**(如「防休眠已结束」),避免用户不知何时恢复。
- **手动关闭**:可随时从托盘关闭,立即生效。

### 3.5 与现有功能的关系
- **番茄钟专注模式**:进入番茄钟时,专注期间**建议自动开启防休眠**(至少保持屏幕常亮),专注结束后恢复到用户此前的防休眠状态。此为体验增强项,可作为 P1。
- **窗口失焦自动隐藏**:防休眠与窗口显隐彼此独立,互不影响。

### 3.6 轻听音乐联动(可选,P1)
若开启「播放轻听音乐时自动防休眠」,则:音乐开始播放 → 自动进入防休眠(仅阻止系统休眠,允许息屏);音乐停止 → 若防休眠是由音乐触发的,则自动解除;若用户手动开过防休眠,则保持不变。

## 4. 交互与状态流转

### 4.1 状态机
```
        点击时长项 / 默认开启
[关闭] ─────────────────────────▶ [已开启(计时中)]
   ▲                                    │
   │  点「关闭」/ 计时到点 / 退出 App      │  切换其他时长 = 重置计时
   └────────────────────────────────────┘
                                         │
                          选「一直保持」 ▼
                              [已开启(无限,手动关闭)]
```

### 4.2 关键规则
- 切换时长(如从 15 分钟改到 1 小时)= **重置计时**,以新时长重新开始。
- 「一直保持」无倒计时,仅手动关闭或退出 App 时解除。
- **退出 App 必须解除所有电源断言**,不得残留导致电脑永不休眠(重要:资源安全)。
- 系统睡眠被用户主动触发(如合盖、手动睡眠)时,遵循系统行为,不强行阻止合盖睡眠。

## 5. 技术实现建议(供研发参考)

> 本节为产品侧的实现方向建议,最终以研发评估为准。

- **原理**:使用 macOS **IOKit 电源断言**(`IOPMAssertionCreateWithName`):
  - 阻止系统休眠:`kIOPMAssertionTypePreventUserIdleSystemSleep`;
  - 保持屏幕常亮:`kIOPMAssertionTypePreventUserIdleDisplaySleep`(即"保持屏幕常亮"开关对应的断言)。
- Tauri(Rust)后端持有断言句柄,前端 / 托盘通过命令开启、关闭;倒计时由后端计时器管理,到点自动释放断言。
- **务必**在 App 退出、崩溃兜底时释放断言(`IOPMAssertionRelease`),防止系统永不休眠。
- 配置随现有设置一起以 JSON 持久化(与 `.todos_settings.json` 同机制,保持隐私本地化)。
- 不调用需要额外权限的接口;防休眠断言无需辅助功能 / 特殊授权。

## 6. 数据与配置项(持久化字段建议)

```jsonc
"keepAwake": {
  "defaultDurationMin": 60,      // 默认时长(分钟);-1 表示"一直保持"
  "keepDisplayOn": true,         // 保持屏幕常亮
  "autoStartOnLaunch": false,    // 开机时自动开启
  "autoOnMusic": false           // 播放轻听音乐时自动防休眠
}
```
> 运行态(是否开启、剩余时间)不持久化,App 重启后除非「开机自动开启」否则默认关闭。

## 8. 边界与异常

| 情况 | 预期处理 |
| --- | --- |
| App 崩溃 / 强退 | 断言随进程结束自动失效,系统恢复正常休眠 |
| 用户手动合盖 | 遵循系统行为,防休眠不阻止合盖睡眠 |
| 同时开启多个时长 | 以最后一次选择为准,重置计时 |
| 自定义时长非法(超范围 / 非数字) | 拦截并提示,恢复上一次有效值 |
| 电池模式(未接电源) | 正常生效,但建议在菜单或提示中弱提醒"防休眠会增加耗电"(P1) |

## 9. 验收标准(Acceptance Criteria)

1. 托盘菜单可开启 / 关闭防休眠,时长选项齐全且当前状态可见。
2. 选定时长后,系统在该时长内不自动休眠;到点自动恢复,并有轻量提示。
3. 「保持屏幕常亮」开关能正确控制是否阻止息屏。
4. 设置页可修改默认时长且立即对下一次「一键开启」生效。
5. 退出 App 后,系统休眠行为完全恢复正常(无残留断言)。
6. 「一直保持」下无倒计时,仅手动关闭 / 退出时解除。
7. 深色 / 浅色主题、多语言下界面与文案均正常。

## 11. 待确认问题(Open Questions)

1. 默认时长定 **1 小时**是否合适?(备选:30 分钟)
2. 防休眠入口是否**仅放托盘**,还是设置页也提供一个手动开关按钮?
3. 托盘状态标识用哪种视觉(圆点 / 咖啡杯 / 眼睛)?需与现有极简托盘风格匹配。
4. 「一直保持」是否需要在开启超过一定时长后给一次「仍在防休眠」的温和提醒,避免用户忘记?
5. 番茄钟联动默认是「开」还是「关」?
