## 防休眠加倒计时（同一日第三轮）
用户指出原实现缺倒计时：`-t` 到点仅后端释放断言，UI 开关不回弹、无倒计时展示。补充：
- 前端倒计时：启用后 `startKeepCountdown()` 每秒刷新「剩余 X分钟/小时/天/周」；到点自动 `enabled=false` 并 `set_keep_awake(false)` 释放（开关回弹）。
- 取消：`#keepawake-cancel` 按钮 = 关开关 = 停定时器 + 释放（复用 change 监听）。
- 重设重置：拖动/点击刻度改档位时若启用中，重新 `invoke set_keep_awake(true,i)` 并 `startKeepCountdown(true)` 以当前时刻重计。
- 持久化新增 `startedAt`（毫秒时间戳），存于 `settings.keepAwake`。
- 后端 `apply_keep_awake_from_settings` 新增 startedAt 过期判断（`std::time::SystemTime` 算毫秒），App 关闭期间已到期则不再拉起，由前端 init 将开关回弹关闭。
- HTML 新增 `#keepawake-countdown`(剩余文本)+`#keepawake-cancel`(取消按钮) 及 CSS；settings.js 顶部加 `elKeepCountdown/elKeepRemain/elKeepCancel` 与 `keepTimer` 单例。
- 永久档（index=9）：不倒计时，仅显示「永久生效」，无取消按钮（取消=关开关）。
- 验证：`cargo check` 与 `node --check settings.js` 均通过（无新增错误/警告）。
