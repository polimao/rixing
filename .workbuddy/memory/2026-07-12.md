## 翻译面板智能化 + 模型自动管理 (src/settings.html, src/settings.js)
1. 智能目标语言：新增 `trDetectTargetLang(text)`，统计 CJK 字符占比，>50% 中文 → 目标 English，否则 → Chinese；后端 prompt 为 `Translate the following segment into {tgt_lang}`，源语言由模型自动检测（原 `#src-lang` 下拉本就未被 JS 使用）。输入时 `trUpdateCharCount` 实时更新 `#tgt-lang-label` 显示检测结果。
2. 移除源语言那一行：删掉 `源语言` 标签 + `#src-lang` 下拉（保留 `→` 箭头 + `#tgt-lang-label` 作为智能检测结果提示）。
3. 移除「加载模型」「下载模型」两个按钮（不显示给用户选择）。翻译按钮点击后由 `trEnsureModel()` 自动保证模型就绪：未下载 → `trStartDownload()`（下载后自动 `load_translate_model`）→ 已下载未加载 → 直接 load；全程透明，无需用户操作。`trToggleModel` 与 `I18N_TO_MT_LANG` 已删除（不再需要）。
4. 单次翻译成功后自动卸载：`translate-result` 事件回调里调用 `trAutoUnload()` → `invoke('unload_translate_model')`，释放内存；下次点击翻译再自动重新加载。
5. `trUpdateButtonStates` 重写：仅依据 `trIsTranslating/trIsDownloading/trIsUnloading` 控制翻译按钮禁用态与文案（翻译中/下载中/翻译），并同步 loading 指示与源文本框禁用。`trStartDownload` 移除对 model/download 按钮的引用。
6. 校验：`node --check` settings.js 通过；grep 确认无 `I18N_TO_MT_LANG`/`trToggleModel`/`elModelBtn`/`elDownloadBtn`/`src-lang` 残留引用。i18n 已有 `translate_downloading` 键（8 语言），无需新增。
